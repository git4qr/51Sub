; Enumeration Tables
; ------------------
;
; Format is:
;
; ENUMTAB <table-num>
;
;  followed by one or more (up to 32)
;
; ENUMDEF <enum-string> = <enum-value>
;
;  where:   <table-num>     is a decimal table number which will be used to
;                           reference this table
;           <enum-string>   is the string (enclosed in double-quotes) to be
;                           used to identify the enumerated value
;           <enum-value>    is the value corresponding to the enumeration-string
;
; Up to 128 Enumeration Tables may be defined
;

;
; DataScreen Definitions
; ----------------------
;
; Format is:
;
; SCREEN <number> <label>
;
;  optionally followed by up to 32 of
;
; HEADING <x> <y> <text>
;
;  where:   <number>    is a screen number in the range 1 to 256 which will be
;                       used to reference this DataScreen
;           <label>     is a short string (enclosed in double-quotes) which
;                       will be used to label a function key for this DataScreen
;           <x>         is the x-coordinate within the DataScreen (in the range 0 to
;                       (<width>-1) ) of the first character of the heading
;           <y>         is the y-coordinate within the DataScreen (in the range 0 to
;                       (<height>-1) ) of the first character of the heading
;           <text>      is text (enclosed in double-quotes) for the header
;
; Field Definitions
; -----------------
;
; Format is:
;
; FIELD <blkid> <fldno> <lab> <scr> <x> <y> <format> [<limits>] <init> [<opfs>] [ENUMS <tabno>]
;
;  where:   <blkid>     is the block-id for the field
;           <fldno>     is the field number within the block
;           <lab>       is the label (enclosed in double-quotes) used to
;                       identify the field on a DataScreen
;           <scr>       is the number of the DataScreen containing the field
;                       if 0 then this field does not appear on a DataScreen
;           <x>         is the x-coordinate within the DataScreen (in the range
;                       0 to (<width>-1) ) of the start of the field label
;           <y>         is the y-coordinate within the DataScreen (in the range
;                       0 to (<height>-1) ) of the start of the field label
;           <format>    is the format of the field value, one of:
;                           BYTE       - 8-bit value in first byte of field
;                           WORD       - 16-bit value in first two bytes of
;                                        field
;                           INTEGER    - 32-bit value
;                           FIXED(a,b) - fixed-point value in first 'a' bits of
;                                        field, with 'b' FLRactional bits
;           <limits>    optional limits of field value, in form:
;                           [<low-lim>,<hi-lim>]
;                           where: <low-lim> is low limit for value
;                                  <hi-lim>  is high limit for value
;           <init>      is the initial value for the field
;           <opfs>      optional C printf style output format string for field
;                       value; if not specified defaults are:
;                               "%s"    for fields with an enumeration table
;                               "%8ld"  for BYTE, WORD and INTEGER
;                               "%8.3f" for FIXED
;                           N.B. remember to specify the 'l' for non-enumerated
;                                BYTE WORD and INTEGER fields as the 32-bit
;                                values used are longs
;           <tabno>     optional number of enumeration table containing
;                       enumeration definitions for the field
;
